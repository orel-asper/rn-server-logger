# Contributing to rn-server-logger

Thank you for your interest in contributing to rn-server-logger! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/rn-server-logger.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `yarn install`

## Development Setup

### Prerequisites
- Node.js 18+ or 20+
- Yarn or npm
- React Native development environment

### TypeScript
This project uses TypeScript with strict mode enabled. Make sure your code:
- Has no `@ts-nocheck` comments
- Compiles without errors in strict mode
- Has proper type definitions

### Code Style
- Run `yarn lint` to check for linting errors
- Run `yarn format` to format code with Prettier
- Follow the existing code style in the project

## Making Changes

1. Make your changes in your feature branch
2. Add or update tests if applicable
3. Update documentation if needed
4. Run `yarn lint` and fix any issues
5. Run `yarn format` to format code
6. Commit your changes with a clear commit message

## Commit Messages

Follow the conventional commits specification:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for formatting changes
- `refactor:` for code refactoring
- `test:` for test changes
- `chore:` for maintenance tasks

Example: `feat: add dark mode support`

## Pull Requests

1. Push your changes to your fork
2. Open a pull request against the main repository
3. Provide a clear description of the changes
4. Link any related issues
5. Wait for review and address any feedback

## Code Review Process

- All submissions require review
- Maintainers may request changes
- Once approved, your PR will be merged

## Questions?

Feel free to open an issue for:
- Bug reports
- Feature requests
- Questions about the code
- Help with contributing
